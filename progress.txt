REASON: H2 2.4.240 (bundled with Spring Boot 4.0.2) does NOT support `INSERT ... ON CONFLICT (conflict-target) DO UPDATE SET` in MODE=PostgreSQL. This is a confirmed H2 limitation — NOT a configuration issue.

What H2 2.4.240 DOES support in PostgreSQL mode:
  - `INSERT ... ON CONFLICT DO NOTHING` (no conflict target, no update)
  - `MERGE INTO t (cols) KEY(pk_col) VALUES (...)` (H2-proprietary upsert, not PostgreSQL-compatible)

What H2 does NOT support (even in MODE=PostgreSQL):
  - `INSERT ... ON CONFLICT (col) DO NOTHING` (with conflict target — also fails)
  - `INSERT ... ON CONFLICT (col) DO UPDATE SET col = ?` (the spec-required syntax)
  - `INSERT ... ON CONFLICT DO UPDATE SET col = EXCLUDED.col` (also fails)
